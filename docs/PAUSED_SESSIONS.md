# Paused Sessions

## Status

Implementation plan for resumable agent sessions using gVisor checkpoint/restore.

Chetter should keep working without gVisor, but non-gVisor sessions are intentionally one-shot: one completed `session_run` per `agent_session`, with no transcript replay or fake resume mode.

## Problem

Chetter currently treats work as one task, one prompt, one container, then teardown. This does not fit feedback loops where one agent creates a PR, another agent reviews it, and the original authoring agent should wake back up to address review feedback.

## Model

An `agent_session` is the long-lived identity and workspace for an agent.

A `session_run` is one prompt execution inside that session.

```text
agent_session sess_1
  session_run run_1 -> creates PR
agent_session -> paused_waiting_review

review feedback arrives

agent_session sess_1
  session_run run_2 -> addresses feedback
```

For non-gVisor runners:

```text
agent_session sess_1
  session_run run_1 -> completed
agent_session -> completed
```

## v1 Rules

- True pause/resume requires gVisor.
- Resume is same-runner only.
- Pause only after a prompt has completed and the agent is idle.
- Preserve the workspace directory in place; do not copy it for v1.
- Store `required_runner_id` on resume work so only the pinned runner can claim it.
- Track PRs/issues/comments with `agent_session_id`, `session_run_id`, and `task_id`.
- Expire paused sessions with TTL-based cleanup.

## Same-Runner Restore

When pausing, store:

- `pinned_runner_id`
- checkpoint path
- workspace path
- container name
- harness session ID

Resume work must be claimable only by the pinned runner:

```sql
WHERE status = 'pending'
  AND (
    required_runner_id IS NULL
    OR required_runner_id = :runner_id
  )
```

If the pinned runner is offline, the session remains paused and the resume run is not claimed elsewhere.

## Workspace Strategy

Chetter bind-mounts host workspaces into containers:

```text
-v {workspace_dir}:/workspace
```

The gVisor checkpoint captures process state, but the bind-mounted files still live on the host. Therefore v1 pairs every checkpoint with the original preserved workspace directory:

```text
gVisor checkpoint directory
+ preserved workspace directory
```

This is safer than tarring files while the process may have open file descriptors, file offsets, deleted-but-open files, mmap state, or temporary files.

Future workspace backends can implement:

```go
type WorkspaceSnapshotter interface {
    Snapshot(ctx context.Context, workspacePath, snapshotID string) (Metadata, error)
    Restore(ctx context.Context, snapshotID, targetPath string) error
    Delete(ctx context.Context, snapshotID string) error
}
```

Likely backends: `preserve-dir`, `tar-zstd`, `btrfs`, ZFS, XFS reflinks, and CSI volume snapshots.

## PR Feedback Flow

1. Authoring trigger creates an `agent_session` and first `session_run`.
2. Agent creates a PR with footer metadata.
3. Webhook artifact discovery maps the PR to session/run/task IDs.
4. The runner checkpoints the gVisor container and preserves the workspace.
5. The session is marked `paused_waiting_review`.
6. Review feedback arrives.
7. Webhook finds the owning `agent_session` by PR artifact.
8. Chetter enqueues a follow-up `session_run` pinned to `pinned_runner_id`.
9. The pinned runner restores the checkpoint and sends the follow-up prompt.
10. The original agent updates the PR branch.

## Footer

Chetter-authored artifacts should include:

```text
---
Generated by [Chetter](https://github.com/flatout-works/chetter)
Session: sess_abc123
Run: run_def456
Task: task_789
Agent: ... | Model: ... | Runner: ...
```

Existing `Task:`-only footers remain valid for task artifact discovery but cannot drive paused-session resume without session ownership.

## Implementation Phases

1. Add `agent_sessions`, `session_runs`, and checkpoint metadata tables.
2. Add `required_runner_id` and enforce runner-affinity claims.
3. Extend artifact tracking with `agent_session_id` and `session_run_id`.
4. Create a one-shot `agent_session`/`session_run` for every submitted task.
5. Add gVisor `pause_after_success` for successful runs.
6. Add manual resume for pinned sessions.
7. Add webhook-driven resume from PR review feedback.
8. Add workspace snapshot backends beyond `preserve-dir`.
