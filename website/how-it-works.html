The simple version

Chetter is a dispatch desk for coding agents.

The server keeps the queue, audit trail, UI, triggers, and GitHub integration. Runners do the risky part: clone repositories, start agent harnesses in isolated workspaces, stream progress, and return results.

1. Request

Someone asks for work

An MCP client, cron trigger, GitHub PR review trigger, or issue trigger creates a task.

2. Queue

The server stores it

The task is persisted in TiDB or MySQL with status pending, team ownership, model settings, timeout, and optional repo metadata.

3. Claim

A runner pulls work

Runners poll over ConnectRPC. One runner claims a task with a database lease so two runners cannot execute the same work.

4. Execute

The agent runs remotely

The runner creates a workspace, clones the repo, writes config, starts the selected harness, and reports events back.

Lifecycle

From prompt to pull request.

This is the normal path for a task that asks an agent to change code and open a PR.

01
Submit

An MCP tool call, webhook, or trigger inserts a task row. The server returns a task id immediately, so callers can watch progress later.

02
Claim

Runners long-poll ClaimTask. Claiming is database-backed with row locking and a 60 second default lease. If no task is available, the runner just polls again.

03
Prepare

The winning runner creates a workspace, optionally clones git_url, writes extra recovery files, injects selected agent and skill definitions, and starts a small workspace MCP server for GitHub actions.

04
Run

The runner starts OpenCode, Claude Code, Pi, or CodeWhale through a harness. In Docker mode it launches an agent container from the configured agent_image; gVisor can add kernel-level sandboxing where supported. OpenCode dispatches prompts asynchronously and polls status, so long tasks survive proxy timeouts and network drops.

05
Report

Progress, heartbeats, token usage, model ids, session ids, summaries, errors, and artifacts flow back through ReportTaskEvents. The web UI updates from persisted task events.

06
Finish or recover

Successful tasks become done. Timeout and transport errors can preserve the workspace and session export, making the session recoverable instead of silently losing context.

Reliability

Leases, not hope

Heartbeats renew leases while work is active. If a runner disappears, the reaper reclaims expired leases back to pending until attempts are exhausted, then fails the task with a timeout category.

GitHub

Artifacts are tracked

Agents create issues, comments, PRs, and reviews through Chetter MCP tools. The server adds canonical signatures and records audit/artifact metadata.

Definitions

Behavior can live in Git

Model catalogs, agents, skills, triggers, and task templates can be synced from a definitions repository. Invalid definitions fail validation instead of silently becoming active.
